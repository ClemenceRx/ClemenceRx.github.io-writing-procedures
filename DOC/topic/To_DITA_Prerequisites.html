Before you start

Be ready to start working on DITA documentation.

Before starting writing with DITA, you need to have some knowledge background with XML, HTML, and CSS.

  • XML - Find XML tutorials on W3Schools website, https://www.w3schools.com/xml/default.asp
  • HTML - Find HTML tutorials on W3Schools website, https://www.w3schools.com/html/default.asp
  • CSS - Find CSS tutorials on W3Schools website, https://www.w3schools.com/css/

Tools Prerequisites

Working with DITA requires the following tools:

  • Java: DITA-OT is designed to run on Java version 8 or later.
  • oXygen XML Editor: the authoring tool to create DITA content and run transformations from XML to the output formats (PDF, HTML). Technical writer in PAX uses version 21.1. All further versions should be compliant with 21.1 version.
  • Gitlab: the version control system for distributed and shared access to versioned content.

Install Java

Download the Oracle JRE or JDK from the Oracle website https://www.oracle.com/technetwork/java/index.html.

Install oXygen XML Editor

You need to install oXygen XML Editor. A 30-days demo version is available at no cost at the following link https://www.oxygenxml.com/xml_editor/download_oxygenxml_editor.html.

Download the latest version of oXygen XML Editor from the oXygen website and launch installation.

Access Gitlab documentation project

To access the DITA files of the PAX Screening Software Documentation, follow the link to the Git documentation project https://gitlab-apc.vanderlande.com/doc/pax-solutions

If you need to create a new project, ask your network administrator to create a new project in Gitlab so that you can store your work in Git and include the documentation in the development release files.

Clone the documentation project from Gitlab

Clone the documentation project to your environment. The following table provides the Gitlab project location corresponding to the document that you need to collaborate on.

Document Location
PAX Checkpoint Screening Software documentation https://gitlab-apc.vanderlande.com/doc/pax-solutions.git
PDA documentation https://gitlab-apc.vanderlande.com/pda/doc.git
Migration Procedure https://gitlab-apc.vanderlande.com/doc/pax-solutions.git

Configure oXygen

Install and configure oXygen XML Editor to be able to build PDF and HTML output of the documentation.

The oXygen publishing templates contain the resources to create PDF documents or HTML Webhelps with the Vanderlande look and feel.
  1. Launch oXygen XML Editor.
  2. In oXygen XML Editor, select Options > Preferences.
  3. In the Preferences window, type "publishing" to filter to the publishing preferences and select Publishing [P].
    The directories to the Additional Publishing Template Galleries open.
  4. Select Add, browse to the Git project that you have cloned and select the publishing templates under oXygen Utilities/[project_name]_publishingtemplate.
    • paxsoftware_publishingtemplate - contains the templates to create PDF and HTML ouputs with the Vanderlande look and feel.
    • migration_publishingtemplate - contains the template to generate a PDF from the markdown file.
  5. Select OK.
    The path to the [project_name]_publishingtemplate directory appears under Directory.
  6. Select OK.
    The publishing templates are now available to create PDF documents or HTML Webhelps with the Vanderlande look and feel.
