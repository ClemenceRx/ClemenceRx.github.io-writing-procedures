Directories

This is the recommended architecture to store your DITA maps and topics.

DITA Maps

Keep DITA Maps (.ditamap) at the root of the documentation directory.

One directory per document

Create a directory for each document.

Each document's directory contains the topics for this document in subdirectories. The subdirectories represent the topic types that you use in DITA:
  • Concept - Contains content that replies to the question "What is it?"
  • Task - Contains content that replies to the question "How do I do it?"
  • Reference - Contains content that replies to the question "What do I need to do it?"
  • Troubleshooting - Contains content that replies to the question "How do I fix it?"
  • Topic - Contains content that cannot be classified in one of the above topic type.

Each subdirectory contains the topics (.dita) files.

Common content

Create a folder for common content:
  • Graphics - To store the images that you use in your documentation.
  • Snippets - To store the topics that are common to several documents (for example, safety rules, disclaimer).
  • Videos - To store the images that you use in your documentation.
