Generate a PDF from a ditamap file

This procedure explains how to generate a PDF from dita topics with oXygen XML Editor.

To generate a PDF, you first have to open the corresponding ditamap file. The following table provides the correspondance between the ditamap file and the document.

Prerequisites
  1. Make sure that you have cloned the documentation project from Gitlab, see Clone the documentation project from Gitlab.
  2. Make sure that you have configured oXygen XML Editor to use the available Publishing Templates. See Configure oXygen

Open the DITA Map file

  1. Open oXygen XML Editor, and make sure that the DITA Maps Manager view is displayed on the left of the window. If not, select Window > Show View > DITA Maps Manager.
  2. From the DITA Maps Manager, select Browse > Open.
  3. Select the .ditamap file corresponding to the PDF file that you want to generate as described in the table.
    The ditamap file opens in the DITA Maps Manager.
  4. Double-click the root item of the ditamap.
    The ditamap file opens in the editor.

Create the PDF Transformation Scenario

  1. Make sure that Root map is set to <Current map>, if not, select <Current map> in the drop-down list.
  2. Select Configure Transformation Scenarios.
    The Configure Transformation Scenarios window opens.
  3. In the Configure Transformation Scenarios window, select DITA Map PDF - based on HTML5 & CSS then select Duplicate.
    The Edit DITA Scenario window opens.
  4. In Name, type a name for the transformation, for example, PAX PDF or PAX HTML Help and in Templates, select PAX Software.
  5. Select Parameters and double-click output.dir.
    The Edit Parameter window opens.
  6. In Value, select Browse and select a location for the resulting PDF file or HTML files, then select OK.
  7. In the Edit DITA Scenario window, select OK.
    The new transformation scenario appears in the list and is automatically selected.
  8. Make sure that the transformation is selected, make sure to close any open PDF files and select Apply Associated.
    The transformation starts and builds the output.
  9. Open the directory defined for the output and check the resulting files.
