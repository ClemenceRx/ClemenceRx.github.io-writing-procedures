Generate a PDF from a markdown file

This procedure explains how to generate a PDF with the Vanderlande look and feel from a markdown file. This procedure applies to the generation of the PDF version of the PAX CHECKPOINT Screening Software migration procedure.

You need to access the migration markdown file available on Gitlab at the following path https://gitlab-apc.vanderlande.com/doc/evelocity-migration/-/tree/develop/src.

Prerequisites
  1. Make sure that you have cloned the documentation project from Gitlab, see Clone the documentation project from Gitlab.
  2. Make sure that you have configured oXygen XML Editor to use the available Publishing Templates. See Configure oXygen

Duplicate the older version of the DITA Map file

  1. In the documentation project, under pax-solutions\src\EN, copy and paste the latest migration_procedure - xx_to_xx.ditamap in order to duplicate the existing ditamap file.
  2. Rename the ditamap file with the new release number.

Open the DITA Map file and update the version number

  1. Open oXygen XML Editor, and make sure that the DITA Maps Manager view is displayed on the left of the window. If not, select Window > Show View > DITA Maps Manager.
  2. From the DITA Maps Manager, select Browse > Open.
  3. Select the migration ditamap file that you have just duplicated and select Open.
    The DITA Map file opens in the DITA Maps Manager view.
  4. Double-click the root item of the ditamap.
    The ditamap file opens in the editor.
  5. At the bottom of the editor view, select the Text tab.
    The content of the ditamap file appears in XML format.
  6. Inside the <booklibrary> tab, modify the numbers to reflect the release version.

Download the migration document content from Gitlab

  1. Open Gitlab, https://gitlab-apc.vanderlande.com/doc/evelocity-migration/-/tree/develop/src and select the last version of the migration procedure, entitled live_x.x_to_x.x.md
    The Markdwon file opens.
  2. Select Download.
    The markdown file is downloaded to your system.
  3. Save the .md file under pax-solutions\src\EN\MIGRATION.

Update the DITA Map file with the reference to the markdown file

  1. Double-click the root item of the ditamap.
    The ditamap file opens in the editor.
  2. In the editor view, inside the topicref tag, modify the reference to the markdown file to refer to the .md file that you have just saved in the MIGRATION directory and save your work.

Create the transformation scenario and build the PDF

  1. Make sure that Root map is set to <Current map>, if not, select <Current map> in the drop-down list.
  2. Select Configure Transformation Scenarios.
    The Configure Transformation Scenarios window opens.
  3. In the Configure Transformation Scenarios window, select DITA Map PDF - based on HTML5 & CSS then select Duplicate.
    The Edit DITA Scenario window opens.
  4. In Name, type a name for the transformation, for example, PAX Migration, and in Templates, select PAX Software Migration PDF.
  5. Select Parameters and double-click output.dir.
    The Edit Parameter window opens.
  6. In Value, select Browse and select a location for the resulting PDF file or HTML files, then select OK.
  7. In the Edit DITA Scenario window, select OK.
    The new transformation scenario appears in the list and is automatically selected.
  8. Make sure that the transformation is selected, make sure to close any open PDF files and select Apply Associated.
    The transformation starts and builds the output.
  9. Open the directory defined for the output and check the resulting files.
